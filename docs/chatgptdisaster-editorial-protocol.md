# ChatGPTDisaster.com Editorial Protocol

Date added: 2026-05-17

## Reddit Testimonial Rule

Any Reddit testimonial, quote, user complaint, or paraphrased Reddit-sourced user report published on ChatGPTDisaster.com must follow this rule:

- Use only real public Reddit posts or comments.
- Include a working source link for every testimonial or source reference.
- Do not fabricate usernames, quotes, screenshots, vote counts, post titles, or user experiences.
- Do not rewrite a Reddit quote as if it were verbatim. If wording is changed, label it as a paraphrase or summary.
- Keep direct quotes short and accurate.
- Omit usernames unless attribution is necessary for verification.
- Review each item for privacy, harassment, medical, legal, mental-health, and defamation risk before publishing.
- Avoid claims that OpenAI or any person directly harmed a specific user unless the cited source clearly states that allegation and the page labels it appropriately.
- Do not use Reddit material from private, deleted, or inaccessible posts as a public testimonial.

This rule applies to homepage modules, story pages, article pages, screenshots, social embeds, static discovery sections, and automated content-generation scripts.

## Pre-Publish Checklist

Before any Reddit testimonial or user complaint goes live:

1. Open the Reddit source and confirm the post or comment exists.
2. Confirm the page copy is either an accurate short quote or a clearly labeled paraphrase.
3. Confirm the source link is present near the testimonial.
4. Remove unnecessary usernames or personal details.
5. Check that the wording does not exaggerate the source.
6. Check that no fake testimonial, fake user, or fake screenshot was introduced.
7. Confirm no canonical tags, SEO metadata, navigation, or unrelated internal links were changed as part of the update.

## Credential Handling Rule

Credentials, secrets, passwords, tokens, FTP URLs containing credentials, and hosting control-panel access details must never be printed, pasted into terminal output, committed, logged, included in diagnostics, uploaded as deployment artifacts, or exposed in screenshots. Diagnostics must use secret names, masked values, or boolean pass/fail checks only.
